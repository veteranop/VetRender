================================================================================
WHAT IT DOES
================================================================================

Adds realistic RF loss based on terrain type:
  • Water bodies: -2 dB (reflective)
  • Urban areas: +6 to +18 dB (buildings block signal)
  • Forests: +4 to +8 dB (foliage absorption)
  • Fields: +1 dB (minimal obstruction)

Data source: OpenStreetMap (free, worldwide)

================================================================================
INSTALLATION
================================================================================

1. Open Command Prompt or PowerShell

2. Install shapely:

   py -m pip install shapely

   OR

   python -m pip install shapely

3. Verify installation:

   py -c "import shapely; print('✓ Shapely installed')"

================================================================================
HOW TO ENABLE
================================================================================

Currently DISABLED to not interfere with your existing calculations.

To enable:

1. Open: gui/main_window.py

2. Find line ~140 where PropagationController is created:

   self.propagation_controller = PropagationController(self.antenna_pattern)

3. Change to:

   self.propagation_controller = PropagationController(
       self.antenna_pattern,
       use_land_cover=True  # ← Add this line
   )

4. Restart VetRender

================================================================================
FIRST RUN
================================================================================

When enabled, first calculation will:
  • Fetch OpenStreetMap data for your coverage area (~30-60 seconds)
  • Cache data locally (land_cover_cache/ directory)
  • Subsequent runs use cache (instant)

You'll see messages like:
  "Fetching land cover data from OpenStreetMap..."
  "Found 234 water, 1,523 urban, 89 forest features"

================================================================================
PERFORMANCE
================================================================================

  • First run (data fetch): +30-60 seconds
  • Cached runs: +0.5 seconds (negligible)
  • Per-point calculation: <0.001 ms

The slowdown is minimal compared to terrain elevation fetching.

================================================================================
CACHE MANAGEMENT
================================================================================

Cache location: land_cover_cache/

To clear cache and force fresh data:
  • Delete land_cover_cache/ directory
  • Or set force_reload=True in code
